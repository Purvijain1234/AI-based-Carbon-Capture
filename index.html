AI-Based Carbon Capture

Optimizing carbon capture operations using AI-driven time-window selection. The system forecasts pollution patterns and operates capture systems only when carbon removal is energy-efficient and effective.

Data Collection Layer

Collects real-time sensor data including CO₂ concentration, PM2.5 / PM10, temperature, humidity, and wind speed. Data is stored and prepared for forecasting future carbon trends.

Prediction Layer – LSTM Model

Core machine-learning component that performs EDA, feature engineering, and time-series training using an LSTM model.

Data Sources

  • Real-time sensor data
  • Open datasets (CPCB / AQI)

Model Pipeline

  • Exploratory data analysis
  • Feature engineering
  • Time-series LSTM training

Forecast Output

Generates weekly carbon concentration trend forecasts, similar to weather reports that show expected pollution levels across time slots.

Decision Engine

Acts as the brain of the system.

Inputs

  • Forecasted CO₂ concentration
  • Energy-efficiency and cost thresholds

Decision Logic

If CO₂ is high and efficiency is acceptable
→ Operate
Otherwise → Do not operate

AI does not capture carbon — it decides when capturing carbon makes sense.

Maintenance & Review

Tracks operational logs, system health, and compares expected versus actual performance.

Monitors

  • Carbon captured per time slot
  • Energy used per time slot
  • Total operational cost
  • Machine health indicators

Detects

  • Efficiency drop
  • Filter cleaning needs
  • Solvent checks
  • Unnecessary start–stop cycles

Alerts

  • Automated alerts (rule-based initially)
  • Prevents long-term efficiency loss
  • Improves future operating decisions
